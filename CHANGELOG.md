# 변경 이력

릴리스마다 `## <버전>` 섹션을 추가하세요. `release.sh`가 해당 섹션을 읽어
Sparkle 업데이트 창과 GitHub 릴리스 노트에 그대로 보여줍니다.

## 1.0.14
- 라이브러리 편집기에 모자이크 도구 추가 — 가리고 싶은 영역을 드래그하면 그 사각형을 모자이크(블록)로 처리. ⌘Z 되돌리기, 복사·저장 시 함께 반영

## 1.0.13
- 캡처 화면을 우클릭(또는 Esc)으로 언제든 취소할 수 있도록 개선
- 단축키 변경이 잘 안 되던 문제 수정 — 녹화 중에는 기존 단축키를 잠시 꺼서 정상적으로 새 조합을 등록
- 저장 폴더를 직접 지정하면 캡처가 그 폴더에 저장되도록 변경 (기본값은 바탕화면/oh-my-opensnap, '기본값' 버튼으로 복귀). 혼동되던 '파일로 저장' 옵션 제거

## 1.0.12
- 캡처 모드의 크로스헤어·화면 어두워짐 오버레이가 OBS 등 외부 화면 녹화에도 보이도록 수정 (앱 자체 캡처 결과에는 여전히 안 찍힘)

## 1.0.11
- 앱 아이콘(로고) 새 디자인으로 교체

## 1.0.10
- 크롭 꼭지점 핸들이 이미지 가장자리에 걸려 잘 안 눌리던 문제 수정 — 이미지 바깥 여백을 눌러도 가까운 핸들이 잡힘

## 1.0.9
- 크롭 범위 조정도 되돌리기(⌘Z) 대상에 포함 — 핸들로 바꾼 범위를 한 단계씩 되돌릴 수 있음
- 크롭의 가운데 핸들(점) 제거 (안쪽을 드래그하면 이동은 그대로)

## 1.0.8
- 크롭 시 핸들을 드래그하면 커서 주변을 확대해 보여주는 확대경(루페) 추가 — 정밀하게 자르기
- 미리보기 가장자리에 여백을 둬 크롭 꼭지점 핸들을 잡기 쉽게 개선

## 1.0.7
- 큰 캡처가 라이브러리 창에 맞게 제대로 축소되지 않던 문제 수정 (맞춤 배율 계산 안정화)
- 이미지 확대/축소 중 화면이 떨리던 현상 개선

## 1.0.6
- 라이브러리 창 크기는 사용자가 맞춘 그대로 유지하고, 캡처 크기에 따라 줌(확대/축소)을 조절해 항상 창에 일관되게 채워 보이도록 변경

## 1.0.5
- 캡처 후 이미지 크기에 맞춰 라이브러리 창 크기를 자동 조절 (너무 크거나 작을 때)
- 업데이트 알림 창에 변경 내용(릴리스 노트) 표시

## 1.0.4
- 자동 업데이트(Sparkle) 동작 검증용 릴리스

## 1.0.3
- 배포 서명을 자체서명 인증서로 변경 — 업데이트해도 화면 녹화 권한이 유지됨

## 1.0.2
- 자동 업데이트 테스트 릴리스

## 1.0.1
- Sparkle 자동 업데이트 도입
- 캡처 진입 시 크로스헤어가 좌상단(0,0)에 잠깐 보이던 문제 수정
