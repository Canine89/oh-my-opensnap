# 변경 이력

릴리스마다 `## <버전>` 섹션을 추가하세요. `release.sh`가 해당 섹션을 읽어
Sparkle 업데이트 창과 GitHub 릴리스 노트에 그대로 보여줍니다.

## 1.0.65
- 설정 창에 화면 녹화 권한 상태를 표시하고, 시스템 설정으로 바로 이동할 수 있게 개선
- 화면 녹화 권한 허용 직후 단축키를 다시 누르지 않아도 캡처가 이어지도록 폴링 추가
- 캡처 후 라이브러리 자동 열기를 설정에서 끌 수 있게 추가
- 신규 설치의 기본 저장 폴더를 Application Support로 바꿔 바탕화면 접근 권한 요청을 줄임(기존 바탕화면 폴더는 유지)
- 루페용 화면 스트림을 축소·저fps로 바꾸고, 커서가 있는 디스플레이만 스트림해 캡처 모드 부하를 줄임
- 영역 이미지 캡처에 sourceRect 경로를 쓰고, PNG 인코딩을 메인 스레드 밖으로 옮겨 캡처 직후 멈춤을 완화

## 1.0.64
- 선택 기능인 브라우저 영역 정밀 감지를 위해 접근성 권한 창을 반복해서 띄우지 않도록 수정
- 화면 녹화 시스템 권한 창과 앱 자체 안내창이 한 번에 연달아 표시되지 않도록 수정
- 업데이트 실패 후 Sparkle 확인 주기를 복구해 다음 자동/수동 업데이트를 다시 시도할 수 있도록 개선

## 1.0.63
- 캡처 직후 클립보드에 TIFF 폴백 대신 저장본과 같은 무손실 PNG만 올리도록 변경
- 일부 앱에 바로 붙여넣을 때 Retina 캡처가 살짝 흐리게 보일 수 있던 경로 제거

## 1.0.62
- 화면 녹화 권한 안내를 보강해 권한 허용 후 앱을 다시 열도록 명확히 안내
- 기존 서명 권한 기록이 남은 사용자를 위해 권한 토글/초기화 안내 추가

## 1.0.61
- 설정 창에 `macOS 로그인 시 자동 실행` 옵션 추가
- macOS 로그인 아이템 상태를 실제 시스템 설정과 동기화해 체크박스에 반영

## 1.0.60
- 창 자동 스냅에서 전체 창이 아닌 윈도우 내부/부분 영역을 이미지 캡처할 때도 윈도우 캡처 원본을 기준으로 잘라 둥근 모서리 알파가 보존되도록 개선
- 자동 스냅 영역을 이동/리사이즈해도 선택이 원래 윈도우 안에 남아 있으면 윈도우 기반 이미지 캡처를 유지하고, 벗어나면 기존 영역 캡처로 처리

## 1.0.59
- 라이브러리 이미지 바깥 클릭이나 Esc로 도구가 해제될 때 상단 도구 메뉴의 선택 상태도 함께 해제되도록 수정

## 1.0.58
- 캡처 선택 HUD를 닫을 때 내용, 그림자, 알파를 먼저 제거해 화면 서버 잔상이 캡처에 남을 가능성을 줄임
- 이미지 캡처와 영상 시작 모두 HUD 닫힘이 반영된 뒤 실행되도록 지연 시간을 조정

## 1.0.57

- 자동 스냅된 전체 창은 둥근 모서리 바깥 배경이 캡처되지 않게 하고, 사용자가 크기/위치를 조정한 선택은 기존 영역 캡처를 유지

## 1.0.56

- 캡처 PNG 저장/복사 시 둥근 모서리 등 투명 영역의 알파가 채워지지 않도록 보존 처리 강화

## 1.0.55

## 1.0.53

- 크롭 모드에서는 가장자리 핸들 보정은 유지하되, 핸들이 아닌 바깥 여백 클릭은 편집 취소로 처리

## 1.0.51

- 한 번 클릭으로 선택, 드래그로 이동, 더블클릭 또는 Return으로 텍스트 편집하도록 동작 정리
- 선택된 텍스트/말풍선에 점선 테두리와 핸들을 표시해 편집 대상을 명확히 구분
- 말풍선 박스 위치를 별도 상태로 저장해 입력 중/완료 후/이동 후 위치가 일관되도록 개선

- 도구 모드와 무관하게 기존 텍스트/말풍선을 클릭해 수정하거나 드래그해 위치를 조정할 수 있도록 개선

- 텍스트 모드에서 빈 곳을 계속 클릭해 새 텍스트를 연속 입력할 수 있도록 수정

## 1.0.47

- 정밀 콘텐츠 영역 계산이 끝나기 전에는 즉시 휴리스틱 영역을 보여주고, 계산 완료 후 조용히 정밀 영역으로 갱신

## 1.0.44

- 접근성 권한이 없거나 `AXWebArea`를 찾지 못할 때 쓰는 Chrome/Safari/Edge/Firefox 계열 fallback 상단 영역 값을 더 현실적인 브라우저 UI 높이에 맞게 조정

## 1.0.42

- 콘텐츠만 선택되는 경우 전체 창 경계는 점선으로, 실제 선택 영역은 강조선으로 표시해 어떤 영역이 캡처될지 더 명확하게 안내

- 선택 영역 바깥만 어둡게 덮어 실제 캡처될 영역이 더 선명하게 보이도록 오버레이 표시 개선

- HUD가 떠 있는 동안에도 핸들로 영역을 계속 수정할 수 있고, 버튼을 누르는 시점의 최종 영역으로 캡처/녹화
